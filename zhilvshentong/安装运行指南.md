# 智旅深瞳项目安装与运行指南

## 环境要求

- Node.js 16.0 或更高版本
- npm 7.0 或更高版本（通常随Node.js一起安装）
- 现代浏览器（Chrome、Firefox、Edge等）

## 安装Node.js

### 方法1：通过官方网站安装（推荐）

1. 访问 [Node.js官方网站](https://nodejs.org/)
2. 下载LTS（长期支持）版本的安装包
3. 运行安装包，按照向导完成安装
4. 安装完成后，重启命令行工具

### 方法2：通过包管理器安装

#### Windows用户（使用Chocolatey）

1. 首先安装Chocolatey包管理器：
   ```powershell
   # 以管理员身份运行PowerShell
   Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
   ```

2. 安装Node.js：
   ```powershell
   choco install nodejs
   ```

#### Windows用户（使用winget）

```powershell
winget install OpenJS.NodeJS
```

### 验证安装

安装完成后，打开新的命令行窗口，运行以下命令验证安装：

```bash
node --version
npm --version
```

如果显示版本号，说明安装成功。

## 项目运行步骤

1. 打开命令行工具，导航到项目目录：
   ```bash
   cd "c:\Users\ZhuanZ\Desktop\软件工程\智旅深瞳\zhilvshentong"
   ```

2. 安装项目依赖：
   ```bash
   npm install
   ```

3. 启动开发服务器：
   ```bash
   npm run dev
   ```

4. 浏览器会自动打开 `http://localhost:3000`，显示智旅深瞳应用。

## 常见问题

### 1. 命令无法识别

如果运行`node --version`或`npm --version`时提示"命令无法识别"，可能是以下原因：

- Node.js未正确安装
- Node.js未添加到系统PATH环境变量
- 需要重启命令行工具或计算机

### 2. 端口被占用

如果3000端口被占用，可以修改vite.config.ts中的端口配置，或者在运行时指定端口：

```bash
npm run dev -- --port 3001
```

### 3. 依赖安装失败

如果`npm install`失败，可以尝试：

```bash
# 清除npm缓存
npm cache clean --force

# 删除node_modules和package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 4. TypeScript编译错误

如果遇到TypeScript编译错误，可以尝试：

```bash
# 重新生成类型定义
npm run build
```

## 生产环境部署

1. 构建生产版本：
   ```bash
   npm run build
   ```

2. 部署dist目录到Web服务器。

## 技术支持

如果遇到其他问题，请检查：

1. Node.js和npm版本是否符合要求
2. 网络连接是否正常（某些依赖需要从国外服务器下载）
3. 防火墙设置是否阻止了npm或Node.js
